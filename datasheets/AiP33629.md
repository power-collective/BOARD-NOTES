## 1. General Description

The AiP33629 is a dot matrix LED drive controller capable of driving common cathode LED dot matrices with constant current. The circuit provides 9 GRID ports and 16 SEG ports, enabling it to drive various sizes of LED panels as needed. The circuit supports 256-level PWM brightness adjustment for each individual dot.

The AiP33629 integrates an IIC communication interface, a constant current control module, display data RAM, an oscillator, and other modules. It can be directly connected to a master controller to achieve display control with minimal external components.

While offering more display functions, its instruction set is compatible with traditional constant voltage LED products such as the AiP1628 and AiP1640, facilitating quick program porting for software engineers and shortening the solution development cycle.

Its main features are as follows:

-   **Drivable Dot Matrix Size:** Maximum 9 GRID × 16 SEG, for common cathode LED matrices; number of GRIDs selectable.
-   **Compatibility:** Instruction set compatible with traditional constant voltage LED products for easy program porting and faster development cycles.
-   **Global Brightness Adjustment:**
    -   Constant current value can be set using the internal reference source via software control (default mode).
    -   Constant current value can be set using an external resistor via software control.
    -   Software-adjustable global constant current control with 64 steps.
-   **Individual Dot Brightness Adjustment:** Each dot supports 256-level PWM brightness adjustment.
-   **Built-in Display Data RAM (DDRAM).**
-   **Communication Interface:** IIC interface.
-   **Built-in Clock Module.**
-   **Built-in Power-On Reset Module.**
-   **Built-in Sleep Mode for Low Power Standby.**
-   **Built-in Blanking Function.**
-   **Operating Voltage:** \( 2.7V \sim 5.5V \)
-   **Package Types:** SOP32/ESOP32/LQFP32/QFN32

## 4. Function Description

## 4.1. Serial Interface

The AiP33629 provides an IIC communication interface with the following features:

-   Two-wire communication: SCL, SDA (SDA is NMOS open-drain output with built-in 5KΩ pull-up resistor).
-   Requires start and stop conditions; restart operation is identifiable.
-   Requires matching slave address.
-   Requires acknowledge bit (ACK) handshake.
-   9 clocks per cycle, MSB first.

#### 4.1.1. Start and Stop Conditions

The AiP33629 detects start and stop conditions when the clock signal (SCL) is high.

#### 4.1.2. Slave Address

The slave address of the AiP33629:

| B7 | B6 | B5 | B4 | B3 | B2 | B1 | B0 |
| :-: | :-: | :-: | :-: | :-: | :-: | :-: | :-: |
| 0  | 1  | 1  | 1  | 1  | CS1 | CS0 | W/R |

Bits B2 and B1 of the slave address are determined by the state of the CS1 and CS0 ports. Therefore, 4 different slave addresses can be set, allowing up to 4 AiP33629 devices to be connected and controlled independently on a single bus.

The lowest bit B0 is the read/write indicator. When W/R \(= 0\), a write operation follows; do not perform read operations.

#### 4.1.3. Communication Format

**Write Data Process:**

[Diagram Placeholder]

M: Master
S: Slave

**Communication Process:**

1.  Master sends START (or reSTART) signal.
2.  Master sends slave address (ensure it matches CS1, CS0 pin states) and W/R control bit (0 for write); slave sends ACK.
3.  Master sends a command; slave sends ACK.
4.  Master sends RAM data; slave sends ACK.
5.  Master sends STOP signal when finished.

The first byte after a START (or reSTART) signal is recognized as the slave address and W/R control bit. When the input slave address matches the circuit, the circuit sends an ACK and communication can continue; when the address does not match, the circuit sends a NAK and ignores all subsequent data.

The second byte of communication is recognized as a command. The AiP33629 cannot write commands continuously. To configure a second command, the current communication must be terminated with a STOP and then restarted with a START, or a reSTART can be used directly. However, it is crucial to complete the 9th clock input (i.e., complete the ACK cycle) before ending the current communication cycle; otherwise, the command from the first 8 clock inputs will be invalid.

When W/R \(= 0\), all data written from the third byte onwards until the STOP is recognized as display data and written to the internal RAM. The address in RAM to which this data is written is determined by the ADS bit in the command.

-   When ADS \(= 0\), data writing always starts from address 0x00, and the RAM address auto-increments by 1 after each byte.
-   When ADS \(= 1\), the starting address for RAM is determined by the ADDR bits in the command, and the RAM address auto-increments by 1 after each byte.

## Helpful Observations (in English)

Based on the translated datasheet, here are some key observations about the AiP33629:

1.  **Dual Brightness Control:** This chip offers a powerful two-tier brightness control. Global brightness is set by adjusting the constant current level (via internal register or external resistor REXT), while *per-dot* brightness is controlled by 256-level PWM data stored in RAM. This allows for very flexible display effects.

2.  **Compatibility & Migration:** A significant feature is its instruction set compatibility with older, constant-voltage drivers like AiP1628 and AiP1640. This is a major advantage for developers looking to upgrade existing designs to constant-current driving for better uniformity and efficiency, as it simplifies software porting.

3.  **IIC Address Configuration:** The CS1 and CS0 pins are not chip-select lines in the traditional sense. They set two bits of the 7-bit IIC slave address. This allows up to four AiP33629 devices to coexist on the same IIC bus without additional hardware, which is excellent for driving large, multi-panel displays.

4.  **Critical Voltage Requirements:** Section 3.2.1 contains a crucial warning about supply voltage. While the chip operates down to 2.7V for logic, the output current capability is significantly impacted by the forward voltage of the connected LEDs. To achieve the set constant current, a higher voltage (4.0V - 5.0V depending on LED color) is required. This must be considered in the power supply design to ensure full brightness.

5.  **Strict Communication Protocol:** The device requires strict adherence to the IIC protocol. Notably, it cannot accept back-to-back command writes without a STOP/reSTART. Also, if the ADS bit is not set, data writes always begin at RAM address 0x00. This is important for proper data sequencing.

6.  **Temperature Protection:** The chip includes built-in temperature protection features (TP1 and TP2). These can be enabled to either reduce the output current or shut down the outputs entirely when the chip exceeds certain temperature thresholds (125°C and 150°C respectively). This is useful for protecting the driver and LEDs in thermally demanding environments.

7.  **Ghost Elimination:** The GS bit in the Display Control command (0x60) enables a "SEG ghost elimination" or blanking function. This likely inserts a short dead time between scanning lines to prevent faintly lit LEDs (ghosting), improving display quality.

8.  **REXT Resistor Value:** When using an external resistor to set the current (RES=1), the formula `Iseg = 1.12 * (17 + I[5:0]) / REXT` is given. A note strongly recommends not exceeding 45mA per SEG pin, suggesting a minimum REXT value of 2KΩ for the maximum current setting (I[5:0]=111111). This is critical for preventing damage to the chip or LEDs.
